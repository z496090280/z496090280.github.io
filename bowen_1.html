rem移动端单位实战

  • 分类：移动端布局
  • 发表于：2017-07-05

这里对rem的原理不做多的解释

  • 个人理解是是一种比例单位
  • 赋值font-size给html
  • 从而使整个html的大小随之变化而变化
  • 支持横竖屏及各种大小屏幕手机
  • 兼容 IE9以上等其他主流浏览器

OK，接下来就看看怎么实际应用吧！

引入 meta 标签

        
            <meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=1.0, user-scalable=no">

这个meta是控制视口大小及缩放

javascript部分

      
        var docEl = document.documentElement;
        function getSize() {
            var screenWidth = docEl.clientWidth;// 获取屏幕的宽度
            docEl.style.fontSize =  screenWidth / (640/60)  + 'px';//设置html字体大小
        }
        getSize();// 页面加载完执行一次
        window.onresize = function() {
            getSize();//窗口或框架被调整大小时触发
        }

这段js控制实际html的font-size值，最后在自己写单位的时候按照设计稿把px全改成rem单位！

番外篇:最后提供一款sublime插件

        
         {
            "px_to_rem":60,
            "max_rem_fraction_length": 4,
            "available_file_types": [".css", ".less", ".sass"]
         }
这款插件名叫cssrem，可自动转换成rem单位。 
第一行表示倍数
第一行表示保留小数点后几位
第三行表示可用的文件类型

总结

这样子就能做到具有“响应式”的效果啦，如果本文有不正确的地方，请指出^_^
